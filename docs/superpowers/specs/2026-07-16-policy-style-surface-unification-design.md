# 情报业务页面政策式表面统一设计

## 背景

当前八个业务页面已经统一为左侧列表、右侧详情的主从工作区，并具备默认打开首条、紧凑字体和高校时间轴等行为。但除政策情报外，其余页面仍大量使用灰色列表底、外层圆角卡片、独立条目边框和多层阴影，导致工具栏、列表、分页与详情被切割成多个视觉容器。

本轮以政策情报页面的一体化排版为视觉基准，统一业务页面的表面、分隔和层级。情报总览不在范围内。

## 目标

- 八个业务页面使用连续白色工作区，形成同一套左侧检索、列表与右侧详情语言。
- 页面标题、数量、更新时间、搜索、筛选和页签继续集中在左上方。
- 右侧详情继续覆盖工作区完整高度，不增加页面级卡片包裹。
- 删除没有独立语义的圆角边框、灰色底块、阴影和列表间空隙。
- 保留舆情指标、履历事件、附件、作者信息等页面特有内容，但以轻量内容分区表达语义。
- 不改变任何数据或交互行为。

## 视觉原则

### 连续画布

业务页面使用白色连续画布。桌面和平板不再显示页面级灰色背景、四周留白、工作区圆角边框或外层阴影。侧栏与业务工作区之间保留现有结构，工作区内部只通过必要的细分隔线建立层级。

移动端沿用现有全屏列表与详情行为。可以保留满足触控识别所需的局部圆角，但不得恢复桌面端的多层卡片堆叠。

### 分隔而非卡片

- 左侧工具栏与列表之间仅保留一条底部分隔线。
- 列表与分页直接相连，分页仅保留顶部分隔线。
- 左右主从区域使用现有垂直分隔线。
- 详情标题区与正文区使用底部分隔线。
- 普通列表条目默认无圆角外框、无阴影、无独立背景，仅用底部分隔线区分。
- 选中条目使用政策情报同源的浅蓝背景和蓝色强调边界，不引入悬浮卡片效果。

### 色彩与字体

- 页面主画布为 `#ffffff`。
- 次级控件可使用 `#f7f8fa` 或 `#f2f4f7`，仅用于页签、筛选或状态背景，不作为整块列表底色。
- 主文字使用 `#1a2b3f` 或现有深蓝文字色，次级文字使用 `#667085`。
- 强调色继续使用现有蓝色体系，不新增主题色。
- 沿用已验收的紧凑字号：页面标题、详情标题、列表正文不放大，社媒情报和高校生态保持当前紧凑密度。

## 共享组件调整

### `IntelligencePageShell`

- 移除 `#f7f8fa` 页面底色。
- 移除桌面和平板的外层内边距与区块间距。
- 保持工作区高度、滚动边界和移动端适配不变。

### `IntelligenceWorkspace`

- 移除圆角、外边框和阴影。
- 保持 40/60 分栏、详情完整高度和 `MasterDetailView` 行为不变。
- 工作区自身只负责尺寸与溢出，不再表现为浮动卡片。

### `IntelligenceToolbar`

- 嵌入模式保持白色背景和底部分隔线。
- 不增加四周边框、圆角或阴影。
- 标题、数量、更新时间、搜索、筛选和页签的位置不变。

### `IntelligenceListItem`

- 默认样式改为平面列表行：无圆角、无四周边框、无阴影。
- 使用底部分隔线和稳定内边距维持扫描节奏。
- 悬停状态只使用轻微底色变化。
- 选中状态使用浅蓝背景，并通过左侧或内侧蓝色边界强化选择；不得改变条目尺寸。
- 焦点环、禁用态和按钮语义保持不变。

### `FeedPagination`

- 移除分页容器圆角、悬浮背景和与列表之间的间隙。
- 仅保留顶部分隔线、白色背景和固定高度。
- 翻页按钮继续使用现有可点击边界与禁用态。

## 页面级列表调整

社媒情报、前沿论文、外部领导、高校生态、两院舆情、两院学术成果和两院专家库的列表容器统一执行以下规则：

- 删除 `bg-[#f7f8fa]`、外围 `p-4` 和条目间 `gap-3`。
- 列表滚动区使用白色背景，内边距只保留内容对齐所需的水平空间。
- 日期时间轴、头像、封面和平台标识继续保留，但不得借助整条圆角卡片形成第二层容器。
- 空状态和加载状态与列表处于同一画布，不放入额外卡片。
- 分页固定在左栏底部并与列表直接连接。

政策情报已有的时间轴和详情结构作为基准，不做无关重写；只确保它与共享表面修改后的边界一致。

## 详情区语义模块

详情正文默认使用连续白色画布和纵向分区。普通段落、元信息和来源引用之间使用留白或细分隔线，不使用嵌套卡片。

以下内容具有独立语义，可以保留明确分区：

- 两院舆情：舆情指标、传播数据、相关账号和媒体附件。
- 外部领导：履历时间轴、任免事件和来源引用。
- 政策情报：政策信息、影响范围和附件。
- 前沿论文与学术成果：作者、机构、论文元数据和原文入口。
- 高校生态：来源、分类、跟踪动作和报告操作。
- 社媒情报：作者信息、正文媒体、互动指标和回复内容。
- 两院专家库：基本信息、研究领域和人才标签。

这些分区使用标题、间距、细分隔线或非常浅的中性底色表达语义。只有图片、视频、附件预览、指标矩阵等天然需要边界的内容可使用轻量边框；不得为每个字段重复创建圆角卡片。

## 行为与数据边界

本轮不修改：

- API 地址、请求参数、缓存和错误处理。
- 搜索、筛选、页签、排序和分页逻辑。
- 桌面和平板默认打开第一条、用户关闭后保持关闭的选择逻辑。
- 移动端列表优先和全屏详情逻辑。
- 高校实际日期分组与时间轴数据。
- 详情原文跳转、焦点恢复、Escape 关闭和滚动行为。

## 响应式规则

- `1440x900`：左栏工具栏、列表和分页连续连接；右侧详情占满工作区高度；页面边缘无灰色沟槽。
- `1280x800`：筛选控件允许换行，但不得形成独立外层卡片；列表行宽度和选中态稳定。
- `1024x768`：沿用现有平板详情覆盖逻辑，表面层级与桌面一致。
- `390x844`：列表与详情分别全屏显示；无横向溢出、文字重叠或底部导航遮挡。

## 验收标准

- 除情报总览外，八个业务页面均无页面级灰色背景、外层圆角工作区和外层阴影。
- 七个非政策页面的普通列表条目不再表现为独立圆角卡片。
- 工具栏、列表和分页在左栏形成连续区域，只使用必要的上下分隔线。
- 右侧详情在所有业务页面均为连续白色画布并覆盖完整高度。
- 选中条目仍具有清晰但克制的浅蓝强调，悬停和键盘焦点可辨识。
- 舆情指标、履历事件、附件等特有模块语义清晰，但不产生多层卡片嵌套。
- 当前已上线的默认详情、高校时间轴、筛选、分页和原文跳转行为全部保持。
- 运行全部 Node 测试、TypeScript 检查和生产构建。
- 对政策情报、社媒情报、高校生态、外部领导、前沿论文和两院舆情执行桌面截图对比，并在 `1024x768` 与 `390x844` 检查响应式布局。

## 非目标

- 不修改情报总览。
- 不新增数据字段、图表、筛选项或业务操作。
- 不重构导航、路由、接口层或状态管理。
- 不把所有页面特有详情强行做成完全相同的内容结构。
- 不以删除所有边框为目标；必要的分隔线、焦点边界和媒体预览边界必须保留。
