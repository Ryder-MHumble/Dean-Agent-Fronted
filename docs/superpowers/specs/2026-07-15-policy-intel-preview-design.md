# 政策情报独立预览页设计

## 目标

新增一个可随时删除的独立实验页面 `/policy-intel-preview`，尽可能复刻用户提供的政策情报参考图。现有政策情报页、全局导航、数据 Hook 和共享组件行为保持不变。

页面同时服务院领导和政策研究人员：首屏直接呈现政策概览、列表、当前政策正文和侧栏信息，用户无需先点击或展开任何模块。

## 隔离边界

- 新页面位于 `app/policy-intel-preview/`。
- 页面专用组件位于 `components/policy-intel-preview/`。
- Banner 图片位于 `public/images/policy-intel-preview-banner.png`。
- 复用现有 `usePolicyFeed`、政策类型和 API 封装，不修改其请求契约。
- 不修改现有 `components/modules/policy-intel/`。
- 不把预览页加入现有侧栏导航。删除上述目录和图片即可完整撤销实验页。

## 视觉结构

### 页面画布

- 使用接近参考图的浅灰白背景，主体最大宽度约 `1800px`，宽屏居中。
- 页面内不显示现有应用侧栏和顶栏，保持参考图的完整独立画布。
- 桌面端页面高度以视口为基准，Banner 与下方工作区共同构成首屏。
- 卡片使用白色背景、细边框、轻阴影和约 `12px` 圆角；不引入全局样式修改。

### Banner

- 高度约占桌面首屏的 `30%`，左侧为标题、副标题和指标条，右侧为生成式视觉资产。
- 标题使用 HTML 文本：`洞悉政策风向`、`把握未来机遇`，第二行重点词使用蓝色强调。
- Banner 图片由 ZGCI Qwen Image 生成，内容为白蓝紫色玻璃球体、细轨道和科技节点，不包含文字，避免生成文字失真。
- 指标条展示真实可得数据：政策总量、政策信源、政策机会、数据更新时间。不存在的数据不使用虚构数值。

### 下方工作区

- 左侧政策列表约占总宽度 `40%`，右侧详情约占 `60%`。
- 右侧详情内部再分为正文区和约 `260px` 的信息侧栏。
- 三个区域均有独立滚动容器，页面整体不因长正文无限增长。
- 首次加载后自动选中当前结果第一条政策，详情始终可见。
- 点击其他政策只替换详情内容，不改变布局尺寸。

## 左侧列表

- 顶部展示 `最新政策`、`高相关度`、`高影响力` 三个排序视图和筛选入口。
- 搜索、分类、日期、信源筛选压缩为紧凑工具栏，保留现有能力。
- 列表项采用参考图的扁平高密度结构，展示标题、来源、标签、日期、相关度和微型趋势线。
- 选中项使用蓝紫描边和极浅背景，其他项使用分隔线，不堆叠独立浮动卡片。
- 分页固定在列表底部。
- `高相关度` 和 `高影响力` 只对当前已加载结果排序，不宣称为全库排序。

## 右侧详情

### 标题区

- 显示标题、来源、发布日期、分类标签、匹配度和原文链接。
- 收藏、订阅、分享、导出等无后端能力的操作不做假交互；仅在有真实行为时显示。

### 正文区

以下模块全部默认展开并按顺序纵向展示：

1. AI 摘要：使用 `aiInsight`，无内容时使用 `summary`。
2. 政策要点：使用 `signals`，为空时不显示列表项。
3. 政策解读：使用 `detail`。
4. 政策原文：使用 `content`。
5. 标签：使用 `tags`。

不使用折叠面板，不要求用户切换 Tab 才能看到已有内容。

### 信息侧栏

以下模块全部默认展开：

- 政策信息：分类、发布机构、发布日期、重要性、资金规模、截止天数。
- 影响范围：仅根据现有字段展示匹配度、相关度和重要性，不推断虚假受众比例。
- 相关附件：类型中没有附件数据时展示简洁空状态，不生成虚假文件名和下载按钮。

文件编号、实施日期、有效期限、浏览量、附件等未知字段不从标题或标签推断。

## 数据和状态

- 加载态使用与最终布局同形的骨架屏，避免首屏跳动。
- 空结果保留 Banner 和筛选区，在工作区显示中文空状态。
- API 失败沿用现有 Hook 的空数据语义，不混入 Mock 政策。
- 筛选或翻页后，自动选择新结果的第一条；无结果时清空详情。
- 页面不新增后端请求类型，不修改真实政策数据。

## 响应式规则

- `>= 1440px`：列表、正文、信息侧栏三列同屏。
- `1024px - 1439px`：列表与详情双列；信息侧栏移动到正文下方并保持全部展开。
- `768px - 1023px`：缩短 Banner，高密度双列；详情内部单列。
- `< 768px`：Banner、指标和列表纵向排列；选择政策后进入独立详情视图，详情中的所有模块仍默认展开，并提供明确返回按钮。
- 不使用整页 `transform: scale()`。通过 CSS Grid、`clamp()`、固定比例和最小宽度控制实现视觉比例一致，保证文字和点击区域可用。

## 生成图片要求

- 使用 `zgci-qwen-image` 的生成脚本。
- 画面为宽幅、无文字、无机器人、无人物、无 UI 截图。
- 主体为半透明蓝紫玻璃球体和白色发光几何标记，周围有多层细轨道、少量蓝色节点和充分留白。
- 整体亮度高、背景接近白色，适合右半幅展示并与 HTML 指标条叠合。

## 验收标准

- `/policy-intel-preview` 可直接访问和刷新，原 `/` 页面行为不变。
- 桌面首屏默认同时看到 Banner、政策列表、政策详情和信息侧栏。
- 所有详情模块默认展开，首次进入不需要点击政策才能看到内容。
- 页面没有虚假政策字段、虚假附件或无效操作按钮。
- `1920x1080`、`1440x900`、`1024x768`、`390x844` 下无水平溢出、文字遮挡或控件重叠。
- 现有 Node 测试继续通过，生产构建通过。
- 使用 Playwright 截图对照参考图检查布局比例、首屏密度和响应式结果。
