# 情报引擎前端优化设计

## 目标

将现有 Dean Agent 前端调整为面向情报查询与数据能力接入的中文 GUI，项目标识统一为 `intelligence-fronted`，保留当前 Next.js 16、React 19、Tailwind CSS 和客户端页面切换架构。

## 范围

- 品牌改为“情报引擎 / 智能创新中心”。
- 顶部移除全局搜索和快速接入菜单，改为“情报引擎信源池”外链。
- 外部导航新增“前沿论文”，并将“科技前沿”改为“社媒情报”、“领导画像”改为“外部领导”。
- 新增“内部共享资讯”，包含“两院学术成果”和“两院专家库”。
- 优化外部领导表格、简介展示、统计布局和头像 URL 覆盖。
- 不移动服务器项目目录；目录改名不属于本轮部署必要条件。

## 信息架构

侧栏保持现有单页应用导航方式，分为三组：

1. 情报总览
2. 外部情报资讯：政策情报、社媒情报、前沿论文、外部领导、高校生态、两院舆情
3. 内部共享资讯：两院学术成果、两院专家库

页面只展示数据标题、筛选项、字段标签和事实性提示，不增加宣传口号、方法说明、英雄区或 CTA 式营销内容。

## 数据架构

### 外部领导

继续使用 `/api/leaders` 的真实分页数据。简介从 `leader_details.text`、`leader_details.summary` 或原始 LLM 画像字段中按优先级提取。表格默认单行截断，完整内容在现有详情抽屉显示。

头像使用前端静态映射补充后端 `leader_details.media.avatar_url`：先读取后端已有 URL，再读取 `public/data/leader-avatars.json`。批处理脚本遍历当前领导数据，优先匹配内部学者库照片，再对缺失记录执行公开搜索。每条结果保留领导 ID、头像 URL、来源 URL和核验状态；未找到时继续显示默认头像。

### 前沿论文

使用 `/api/papers`，按来源和会议分为顶刊、顶会、ArXiv及其他。每次只取有限分页数据，展示标题、作者、摘要、会议或期刊、年份和原始链接。页面提供用户指定的情报引擎 Skill 地址，不尝试在浏览器加载全部 10 万余篇论文。

### 两院学术成果

使用 `/api/papers?source_type=academy_weekly_signature_achievements`，按发表时间倒序展示学生与导师相关论文成果。页面展示作者、论文、期刊、发表日期和来源链接，并提供 `zgca-paper-author-query` Skill 地址。

### 两院专家库

从指定钉钉 AI 表格生成脱敏快照，只允许输出姓名、工作单位、二级单位、专业技术职称、职务或人才称号、地区或国家、研究领域或研究方向、学科方向和更新时间。邮箱、电话、学号及其他敏感字段不得进入前端文件或页面。

页面提供更多学者数据地址以及 `liangyuan-expert-recommender` Skill 地址。快照包含同步时间，并通过脚本重新生成，不在组件中手写专家数据。

## 页面布局

- 保持白底、高密度数据报告样式。
- 外部领导页面删除“领导画像库”及解释文本。
- 总量、待核验数和数据更新时间并入筛选工具栏右侧；移动端移到分页区域，不单独占用一行。
- 论文和成果使用紧凑列表，不使用营销卡片或大标题。
- 专家库使用可筛选表格，敏感字段在数据生成阶段即被排除。
- 所有可点击 Skill 和数据来源使用普通文本链接或轻量提示条。

## 错误与空状态

- API 请求失败时显示现有风格的内联错误或空状态，不伪造数据。
- 头像加载失败时回退到姓名首字或默认人物图标。
- 脱敏快照为空时说明当前没有可展示记录，并保留数据源链接。
- 外部链接统一新窗口打开并设置 `rel="noreferrer"`。

## 测试与验收

- 为论文分类、专家脱敏、领导简介和头像合并逻辑编写 Node 单元测试。
- 先验证测试失败，再实现最小代码使其通过。
- 运行全部 `tests/*.test.mjs`、TypeScript 检查和 `npm run build`。
- 在 1440px 桌面和 390px 移动视口检查侧栏、顶部链接、表格截断、分页和空状态。
- 部署到 `http://10.1.132.21:8080` 后逐项核验所有导航和外链。

## 非目标

- 不建立新的后端数据仓库或钉钉长期同步服务。
- 不重写现有 UI 框架或切换路由体系。
- 不保证所有领导都能找到可信头像；要求是对全部当前记录执行搜索尝试，并保留可追溯结果。
- 不展示专家表中的邮箱、电话或其他私密信息。
